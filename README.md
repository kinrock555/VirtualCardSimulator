# Virtual Card Test

オリジナルカードゲームの制作者向けに、カード画像の登録・デッキ編集・3Dテーブルでのテストプレイをブラウザだけで行える最小構成プロトタイプです。Tabletop Simulator のような自由な操作感を、カードゲームの試作に必要な範囲だけに絞って実装しています。

このアプリはカードゲームの**ルールを一切処理しません**。ターン管理・コスト計算・勝敗判定などはすべてユーザー自身が行う前提です。アプリが管理するのは「カード画像」「デッキデータ」「3Dテーブル上のカードの状態」だけです。

## 起動方法

```bash
npm install
npm run dev
```

表示されたローカルURL（例: `http://localhost:5173`）をブラウザで開いてください。

- `npm run build` : 型チェック(`tsc -b`) + 本番ビルド
- `npm run preview` : ビルド結果のプレビュー

## カード画像の追加方法

1. `src/assets/cards/` フォルダに **PNGまたはJPEG(.jpg/.jpeg)形式** の画像を追加します（例: `card-005.png`）。
2. カード裏面は `src/assets/card-back.png`（または `.jpg`/`.jpeg`）を差し替えます（全カード共通の1枚）。
3. `npm run dev` を再起動する（またはVite開発サーバーがホットリロードで検知)と、BOX画面に自動で一覧表示されます。

- 対応形式は **PNG / JPEG** です。AI/PSD/PDFには対応していません。
- カードIDは画像パス（ファイル名）から自動生成され、BOX画面でカード名を変更してもIDは変わりません。
- カード画像フォルダが空でもアプリはクラッシュせず、「カードが見つかりません」という表示になります。
- 動作確認用に `card-001.png`〜`card-004.png` と `card-back.png` はプレースホルダー画像（色分けされた図形のみ）を同梱しています。実際の制作物では差し替えてください。

## BOXの使い方

`/box` 画面はカード原本ライブラリです（3Dテーブルは表示されません）。

1. カード名で検索できます。
2. カードをクリックすると選択され、右側に拡大プレビューが表示されます。
3. プレビュー下の入力欄でカード名を編集し、フォーカスを外すと保存されます（`localStorage`）。
4. リロードしても変更したカード名は保持されます。
5. 画像そのものの削除や、所持枚数の在庫管理は行いません（BOXは「使えるカードの一覧」です）。

## デッキの作り方

`/decks` 画面で行います（3Dテーブルは表示されません）。

1. 「新規デッキ」でデッキを作成、または上部のドロップダウンから既存デッキを選択して編集します。
2. 左側のBOX一覧から「追加」を押すとデッキに1枚追加されます。同じカードを何度でも追加できます。
3. 右側でデッキ名の変更、各カードの `+` / `-`（1枚単位の増減）、「外す」（そのカードをデッキから完全に削除）ができます。
4. 「保存」を押すまでデッキ名・枚数の変更は確定しません（`localStorage`に保存）。
5. 「複製」「削除」は選択中のデッキに対して行われます。
6. デッキ枚数に上限・下限はありません（1枚デッキ・100枚超デッキも保存可能）。

## テストプレイの開始方法

デッキ編集画面で編集中のデッキの「テストプレイ」ボタンを押すと `/play/:deckId` に遷移します。
デッキに含まれる全カードが、テーブル上に裏向きの山札として生成されます（1枚ずつ独立した`CardInstance`）。

## カードの操作方法

- **左クリック**: カードを選択（黄色い枠で強調 + 画面右上に拡大プレビュー表示）
- **左ドラッグ**: カードを掴んで移動。ドラッグ中は少し浮き上がり、テーブル範囲外には出せません。
- **右クリック**: コンテキストメニューを表示（表向き/裏向き切替、左右90度回転、テーブルから取り除く）。実行できない項目（既に表向きの場合の「表向きにする」など）は無効表示になります。
- **マウスホイール**: ズーム（最小・最大距離あり）
- **右ドラッグ / 中ボタンドラッグ**: 視点回転（左クリックはカード操作専用のため、カメラ操作とは競合しません）
- **カメラ初期位置へ戻すボタン**: 初期視点に戻します。

カードのドラッグ中・コンテキストメニュー表示中はカメラ操作が自動的に無効化されます。

## 主要ファイル構成

```
src/
├─ assets/                 # カード画像置き場（cards/*.png, card-back.png）
├─ types/                  # CardMaster / DeckData / CardInstance などの型定義
├─ lib/                    # id生成・storage・カード読み込み・3D用定数/座標計算
├─ store/                  # Zustandストア(カードマスター/デッキ/テーブル/UI)
├─ components/
│  ├─ common/              # Button, ScreenHeader, ErrorBoundary, 通知バナー
│  ├─ box/                 # BOX画面のグリッド・プレビュー
│  ├─ deck/                # デッキ編集画面の行コンポーネント
│  └─ table/               # 3Dテーブル(R3F): カード表示・ドラッグ・右クリックメニュー・カメラ
├─ routes/                 # MainMenuPage / BoxPage / DeckEditPage / PlayPage
├─ App.tsx                 # ルーティング定義
└─ main.tsx                # エントリーポイント
```

状態管理は4つのストアに分離しています。

- `useCardMasterStore` : カード原本データ + 名前上書き（`localStorage`永続化）
- `useDeckStore` : デッキのCRUD（`localStorage`永続化）
- `useTableStore` : テストプレイ中のカードインスタンス座標・表裏・回転・選択状態（**永続化なし**）
- `useUiStore` : 検索欄の入力値・通知バナーなど画面固有のUI状態

3Dオブジェクト自体はストアに保存せず、座標(`x,y,z`)・回転(`0/90/180/270`)・表裏(`boolean`)のみを保存しているため、将来オンライン同期に置き換える際もデータ層とレンダリング層を分離したまま拡張できます。

## 現在の制限事項

- ルール処理（ターン、コスト、ダメージ、勝敗判定など）は一切ありません。
- 山札からの手札生成・ドロー・シャッフル・山札の中身閲覧はありません（テーブルに全カードが裏向きで並ぶのみ）。
- 複数選択・範囲選択・カードの束（スタック）作成/解除・重ね置きの自動認識はありません。
- 盤面の保存・読み込み・リセット機能はありません（テストプレイ画面を離れると盤面状態は破棄されます）。
- サイコロ・コイン・トークン・カウンターなど補助要素はありません。
- オンライン対戦・ルーム・チャットなどのマルチプレイ機能はありません。
- スマートフォン対応はしていません（PCブラウザ専用）。
- カード画像の差し替え・追加はファイルシステムへの手動配置のみで、UIからのアップロードはできません。

## 既知の不具合・注意点

- 右クリックで「カードを掴んだままドラッグして視点回転しようとする」操作をすると、カード側の`contextmenu`イベントが先に発火しコンテキストメニューが開くことがあります（右ドラッグでの視点回転は、カードのない場所から始めてください）。
- デッキ編集画面で「保存」を押す前に別デッキへ切り替える、または画面を離れると、未保存の変更は失われます。
- 大量のカード画像・大きなデッキ（数百枚規模）を読み込んだ場合のパフォーマンスは未検証です。

## 次に実装すべき機能

- 手札・ドロー・シャッフルなど、山札を実際に運用するための最小限の操作
- 複数選択によるカードの一括移動
- 束（スタック）の作成・解除
- 盤面の保存/読み込み（テストプレイのセッションを永続化）
- BOXへの画像アップロードUI（現状はファイル配置のみ）
- サイコロ・トークンなどの補助オブジェクト
- オンラインでのテーブル共有（現在の状態構造はこれを見据えて座標データのみを保持する設計にしてあります）
