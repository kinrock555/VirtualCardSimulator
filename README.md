# Virtual Card Simulator

オリジナルカードゲームの制作者向けに、カード画像の登録・デッキ編集・3Dテーブルでのテストプレイをブラウザだけで行えるツールです。

## コンセプト

「Tabletop Simulatorの自由な操作感を、カードゲーム制作とテストプレイに特化して簡略化したブラウザツール」

本アプリは以下の独立した画面で構成されています。BOXと3Dテーブルは同じ画面には表示しません。

1. **メインメニュー** - 各画面への入口
2. **BOX** - カード画像の原本を管理する専用画面
3. **デッキ編集** - BOXのカードからテスト用デッキを組む専用画面
4. **3Dテストプレイ** - 作成したデッキを3Dテーブル上で操作する専用画面

このアプリはカードゲームの**ルールを一切処理しません**。ターン管理・フェーズ管理・コスト計算・ダメージ計算・勝敗判定などはすべてユーザー自身が行う前提です。アプリが管理するのは「カード画像」「デッキデータ」「3Dテーブル上のカードの状態（山札・手札・場・束）」だけです。第4段階として、**2人用のミニマルなオンライン対戦モード**を追加しました（詳細は後述の「オンライン対戦モード」を参照）。第5段階では、実際にテストプレイする際の見やすさ・操作性を改善しました。手札は画面下部の専用2D UIへ変更し、3Dテーブルは厚み・フレーム・脚のある立体的なデザインへ、フィールドは明るく見やすく、テーブルの周囲には選択可能な部屋背景（無地・おうち・カジノ・カードショップ）を追加しています。第6段階では、複数枚のカードを続けて場に出しても表示が崩れないよう修正し、表向き/裏向きを選んで場に出せるようにし、カード拡大プレビューが手札パネルの高さに関わらず常に画面内に収まるよう改善し、斜め視点/真上視点を切り替えられるようにし、固定の墓地・除外エリアを廃止して**フィールドはすべて自由配置**に統一し、さらに**1台のPCを2人で交代しながら使うオフライン対人テストプレイ（共通デッキ/個別デッキの3方式、プレイヤー交代、相手の手札枚数表示）**を追加しました。第7段階では、裏向きカードを出した本人だけがこっそり確認できる機能、カードプレビューのマウス追従ルーペ、左右上部と手札の各UIパネルの個別の折りたたみと盤面を最大限広く使う集中モード、2人プレイでのプレイヤーごとのカード初期向きとデッキ初期位置（各プレイヤーの右側）、自分の画像を使えるカスタムプレイマット、軽量/標準の3D品質設定、テーブルと部屋（巾木・ドア・窓・天井照明・テーマごとの背景オブジェクト）の見た目の底上げ、そして3D背景を再利用したゲームらしいメインメニュー（保存データがあれば「前回の続きから」も利用可能）を追加しました。1人用のオフライン機能・オンライン対戦モードの既存機能はすべてそのまま利用できます。

## 現在実装されている機能

- カード画像の自動読み込み（PNG / JPEG）とBOXでの一覧・検索・拡大プレビュー・名称編集
- デッキの新規作成・保存・複製・削除・名称変更、同名カードの無制限追加、複数選択による一括追加
- 3Dテーブル上でのカード選択・ドラッグ移動・表裏切替・90度回転
- 山札（メインデッキ）・カスタム束が同時に複数存在できる「束（スタック）」データ構造。束以外のカードはすべてフィールド上に自由配置されます（固定の墓地・除外エリアはありません。詳細は後述）
- 山札の中身を2D一覧で確認・検索・ドラッグ並び替え・各ゾーンへの移動
- シャッフル、1枚ドロー、任意枚数ドロー（クイックボタンつき）、一番上をめくる
- Shiftクリックによる複数選択と、選択カードの一括移動・表裏・回転・ゾーン移動
- 選択カードから明示的に束を作成し、シャッフル・ドロー・中身確認・解除ができる
- **画面下部の2D手札パネル**（第5段階で刷新）: 手札は3D空間ではなく専用のフラットUIに表示され、カメラを動かしても位置は変わりません。横一列表示、枚数が多い場合の重ね表示と横スクロール、ホバーで持ち上がる演出、選択中カードの強調、折りたたみ可能
- **手札⇔フィールドの移動**: 手札カードを左ドラッグすると表向きで、**Shift＋左ドラッグすると裏向きで**フィールドへ出せます（第6段階で追加）。ドラッグ中のプレビューは選んだ向きに応じて表/裏の見た目に切り替わり、置かれた瞬間から一貫してその向きで表示されます。フィールドカードを手札エリアへドラッグして戻す、右クリックメニューから戻すことも可能です。2枚以上を続けて場に出しても、先に置いたカードが消えたり上書きされたりしません（各カードは独立したインスタンスとして管理されます）
- テーブルリセット（確認ダイアログ付き、全カード回収＋シャッフル）
- 盤面（カードの配置・山札・手札・束・テーマ・部屋背景・カメラ視点・カメラ位置、2人プレイ時はプレイヤー情報・デッキ方式も含む）の名前付き保存/読み込み/名前変更/削除
- カメラのズーム・回転・初期位置リセット（カード操作との競合防止つき）、**斜め視点/真上視点の切り替え**（第6段階で追加、詳細は後述）
- 手札パネルの高さに関わらず常に画面内に収まる、大型化したカード拡大プレビューと、折りたたみ可能な操作方法パネル
- 3種類のテーブルテーマ（切替は即時反映、`localStorage`に保存され再読み込み後も維持）
- **立体的な3Dテーブルと部屋背景**（第5段階で追加）: 厚みのあるテーブル天板・外周フレーム・4本脚、明るく見やすくなったプレイマットと照明、テーブルを取り囲む部屋背景を4種類（無地・おうち・カジノ・カードショップ）から選択可能。テーブルテーマとは独立して切り替えられ、`localStorage`に保存され再読み込み後も維持
- **1台のPCで交代しながら遊ぶ2人用オフラインテストプレイ**（第6段階で追加）: 共通デッキ/個別デッキ（同じ構成・別構成）の3方式、プレイヤー交代（ボタン必須・Tabは補助）、相手の手札は枚数のみ表示（中身は見えません）。詳細は下記「2人用オフラインテストプレイ」を参照
- **裏向きカードの非公開確認**（第7段階で追加）: 自分が裏向きで出した（または裏向きにした）カードだけ、盤面上は裏向きのまま右側プレビューで中身をこっそり確認できます。相手のカードや、相手が裏向きにしたカードは確認できません。詳細は下記「裏向きカードの非公開確認」を参照
- **カードプレビューのルーペ**（第7段階で追加）: 右側のカードプレビュー上でマウスを動かすと、その位置を約2.5倍に拡大表示します。設定でON/OFF切り替え可能。詳細は下記「カードプレビューのルーペ」を参照
- **UIパネルの個別折りたたみと集中モード**（第7段階で追加）: 左側の操作方法パネル・右側のカードプレビュー・上部の操作バー・手札パネルをそれぞれ個別に折りたためます。「集中モード」でこれらをまとめて最小化し、3Dフィールドを最大限広く使えます。詳細は下記「UIパネルの折りたたみと集中モード」を参照
- **プレイヤーごとのカード向きとデッキ位置**（第7段階で追加、2人プレイ時）: 手札から場に出したカードは、出した瞬間のプレイヤーに応じて0度/180度の向きで配置されます。各プレイヤーの山札は、それぞれの座席から見て右側に初期配置されます。詳細は下記「プレイヤーごとのカード向きとデッキ位置」を参照
- **カスタムプレイマット**（第7段階で追加）: PNG/JPG/JPEG/WebP形式の自分の画像をプレイマットとして登録・選択・削除できます（ブラウザ内のIndexedDBにのみ保存）。詳細は下記「カスタムプレイマット」を参照
- **3D品質設定・部屋の見た目向上**（第7段階で追加）: 設定画面から「標準」/「軽量」の3D品質を選べます。テーブル・部屋（巾木・ドア・窓・天井照明・テーマごとの背景オブジェクト）の見た目も改善しています。詳細は下記「3D品質設定」「部屋テーマの改善」を参照
- **ゲームらしいメインメニュー**（第7段階で改善）: 実際の3Dテーブル・部屋を軽量設定で背景に再利用し、ガラス風パネルのデザインへ刷新。保存済み盤面がある場合は「前回の続きから」で直前の状態を再開できます。詳細は下記「メインメニュー」を参照
- **オンライン対戦モード（2人用・第4段階で追加）** : ルーム作成・招待URLでの参加、カード移動/表裏/回転/ゾーン移動/シャッフル/ドロー/束操作/テーマ変更/テーブルリセットの同期、手札の非公開（相手には枚数と裏面のみ表示）、再接続時の自動復帰と手札復元。詳細は下記「オンライン対戦モード」を参照。

## 起動方法

### オフライン（1人用）のみで使う場合

```bash
npm install
npm run dev
```

表示されたローカルURL（例: `http://localhost:5173`）をブラウザで開いてください。オンライン対戦モードのボタンは表示されますが、オンライン用サーバーを起動していない場合はルーム作成・参加時に接続エラーになります。

### オンライン対戦モードも使う場合

オンライン対戦にはクライアント（Vite）とは別に、ローカルのオンライン用サーバー（Socket.IO）を起動する必要があります。

```bash
npm install
npm run dev:online
```

`dev:online` はクライアント（`http://localhost:5173`）とオンラインサーバー（`http://localhost:3001`）を同時に起動します。個別に起動したい場合は次のコマンドも使えます。

- `npm run dev:client` : クライアント（Vite）のみ起動
- `npm run dev:server` : オンライン用サーバー（`tsx watch server/index.ts`）のみ起動、既定ポートは `3001`（環境変数 `PORT` で変更可）
- `npm run build` : クライアントの型チェック(`tsc -b`) + 本番ビルド + サーバーの型チェック(`typecheck:server`)
- `npm run typecheck:server` : サーバー側だけの型チェック（`tsc -p server/tsconfig.json`、実行ファイルは出力しません）
- `npm run lint` : `oxlint`（`server/`・`shared/`も対象）
- `npm run preview` : クライアントのビルド結果のプレビュー

オンライン用サーバーはヘルスチェック用に `GET /health` を提供します（`{"ok":true,"rooms":<現在のルーム数>}` を返します）。

## 使用技術

- TypeScript / Vite / React / React Router
- Three.js / React Three Fiber / @react-three/drei
- Zustand（状態管理）
- ブラウザ標準の `localStorage`（永続化。IndexedDBは未使用）
- **オンライン対戦モード**: Node.js + TypeScript製の最小構成サーバー（`node:http` + `socket.io`）、クライアント側は`socket.io-client`。ルーム状態はすべてサーバーのメモリ上（`Map`）のみで管理し、データベースは使用していません。
- 外部データベース・外部ストレージ・外部認証・外部AIサービス・外部SaaS（Firebase/Supabase/Pusher/Ablyなど）は一切使用していません

## 有料サービスについて

本プロジェクトは、有料API・有料クラウド・有料データベース・有料ストレージ・有料認証・有料素材のいずれも使用していません。オンライン対戦モードを含め、すべてローカル環境と無料のオープンソースライブラリ（Socket.IOなど）、ブラウザ標準機能のみで動作します。外部のSaaS/BaaS（Firebase、Supabase、Pusher、Ablyなど）は無料枠であっても使用していません。

## BOXの使い方

`/box` 画面はカード原本ライブラリです（3Dテーブルは表示されません）。

1. カード名で検索できます。
2. カードをクリックすると選択され、右側に拡大プレビューが表示されます。
3. プレビュー下の入力欄でカード名を編集し、フォーカスを外すと保存されます（`localStorage`）。
4. リロードしても変更したカード名は保持されます。
5. 画像そのものの削除や、所持枚数の在庫管理は行いません（BOXは「使えるカードの一覧」です）。

## デッキ編集の使い方

`/decks` 画面で行います（3Dテーブルは表示されません）。画面は左（BOXのカード一覧）と右（編集中のデッキ）に分かれています。

1. 「新規デッキ」でデッキを作成、または上部のドロップダウンから既存デッキを選択して編集します。
2. 左側のBOX一覧から「追加」を押すと、そのカードを1枚デッキに追加できます。
3. 右側でデッキ名の変更、各カードの `+` / `-`（1枚単位の増減）、「外す」（そのカードをデッキから完全に削除）ができます。
4. 「保存」を押すまでデッキ名・枚数の変更は確定しません（`localStorage`に保存）。
5. 「複製」「削除」は選択中のデッキに対して行われます。
6. デッキ枚数に上限・下限はありません（1枚デッキ・100枚超デッキも保存可能）。

### 一括選択の使い方

- 左側のカード一覧では、チェックボックスまたはカード本体のクリックで複数カードを選択できます。選択中のカードは枠で強調されます。
- 「表示中をすべて選択」は、現在の検索結果として表示されているカードだけを対象にします。
- 「選択解除」で選択状態をクリアできます。
- 「枚数」欄で1以上の整数を指定し、「選択カードを一括追加」を押すと、選択した全カードがその枚数ずつデッキへ追加されます（デッキに同じカードが既にある場合は加算されます）。
- 一括追加後、選択状態は自動的に解除されます。選択状態自体はUI上だけの一時的なもので、デッキデータには保存されません。

## テストプレイの始め方

**1人でテストプレイする場合**は、デッキ編集画面で編集中のデッキの「テストプレイ」ボタンを押すと `/play/:deckId` に遷移します（この起動方法はこれまで通りです）。デッキに含まれる全カードが、裏向きの山札（メインデッキ）として3Dテーブル上に生成されます（1枚ずつ独立した`CardInstance`）。

**2人でテストプレイする場合**は、メインメニューの「2人でテストプレイ」からデッキ方式・プレイヤー名・使用デッキを選んで開始します。詳細は下記「2人用オフラインテストプレイ」を参照してください。

画面は上から「3Dフィールド（テーブル・部屋背景）」と「画面下部の手札パネル」の2段構成です。手札パネルはカメラを動かしても常に画面下部に固定表示されます。

## 手札エリアの使い方（画面下部の2D UI）

手札は3D空間には配置されず、画面下部に固定された専用のフラットUIに表示されます（Tabletop Simulatorの手札管理に近い方式ですが、見た目は本アプリ独自のものです）。

- 見出しに「手札」と現在の枚数が表示されます。
- 手札カードは横一列に並びます。枚数が少ないときは間隔を空けて中央に、10枚を超えると少しずつ重ねて表示され、画面幅に収まらない分は横スクロールで確認できます。
- カードにマウスを重ねると少し持ち上がり、選択中のカードには枠と影が付きます。
- 右上の「▼たたむ / ▲開く」ボタンで手札パネルを折りたたみ・展開できます（開閉状態は`localStorage`に保存され、再読み込み後も維持されます）。
- 3Dフィールドと手札パネルの境界は、背景色の違い・上部の境界線・影で明確に分けています。

### 手札からフィールドへ出す

手札パネルのカードを**左ドラッグ**すると、半透明のプレビューがマウスに追従し、そのカードは**表向き**でテーブルに置かれます。**Shiftキーを押しながら左ドラッグ**すると、同じ操作で**裏向き**に置けます。表/裏はドラッグを開始した瞬間に決まり、ドラッグ中のプレビュー（マウスに追従する半透明画像・3Dフィールド上の黄色/灰色の設置プレビュー）にもその向きがそのまま反映されるため、置く前に見た目で確認できます。フィールド上でマウスを離すとそのカードが選んだ向きでテーブルに置かれ、手札から取り除かれます。手札パネルの中で離した場合や、3Dフィールドの外で離した場合はカードが手札にとどまり、消えることはありません。置いた後に向きを変えたい場合は、通常通りカードを右クリックして「表向きにする/裏向きにする」を選べます。

### フィールドから手札へ戻す

以下の2通りの方法があります。

1. カードを右クリック →「手札へ戻す」（確実に動作する方法です）
2. フィールド上のカードを画面下部の手札パネルへ左ドラッグする

いずれの方法でも、カードは表向きになり手札の末尾に自動整列します。

## カードの操作方法

- **左クリック**: カードを選択（黄色い枠で強調 + 画面右側に拡大プレビュー表示）
- **Shift + 左クリック**: 選択中のカードを維持したまま追加/解除（複数選択）。すでに選択中のカードをShift+クリックすると選択から外れます。
- **Escキー / 何もない場所を左クリック**: 選択をすべて解除します。
- **左ドラッグ**: フィールド上のカードを掴んで移動。複数選択中は選択カード全体の相対位置を保ったまま一括移動します。ドラッグ中は少し浮き上がり、テーブル範囲外には出せません。手札パネルへドラッグすると手札へ戻ります（上記参照）。
- **右クリック（通常カード）**: カードメニューを表示（表向き/裏向き切替、左右90度回転、手札へ戻す、山札の上/下へ戻す、テーブルから取り除く）。
- **右クリック（複数選択中）**: 一括操作メニューを表示（表向き/裏向き、回転、手札へ戻す、束にする、選択解除）。
- **右クリック（山札・束）**: それぞれ専用メニューを表示します。
- **マウスホイール**: ズーム（最小・最大距離あり）
- **右ドラッグ / 中ボタンドラッグ**: カメラ視点の回転（左クリックはカード操作専用のため、カメラ操作とは競合しません）
- **画面上部の「カメラ初期位置へ戻す」**: 現在選択中の視点（斜め/真上）の初期位置に戻します。
- **Vキー、または画面上部の「視点：斜め/真上」ボタン**: 斜め視点と真上視点を切り替えます（下記「カメラ視点の切り替え」を参照）。文字入力欄にフォーカスがある間や、メニュー/モーダル表示中は無効です。

カードのドラッグ中・各種右クリックメニュー表示中はカメラ操作が自動的に無効化されます。左側の「操作方法」パネルは折りたたみ可能で、初期状態では開いています（開閉状態は`localStorage`に保存されます）。

## カメラ視点の切り替え（斜め視点・真上視点）

3Dフィールドのカメラには2つのプリセット視点があります。

- **斜め視点（初期状態）**: 従来通りの見下ろし気味の視点で、カードの手前/奥の関係や立体感を確認できます。
- **真上視点**: テーブルをほぼ真上から見下ろす視点です。テーブル全体が画面に収まり、カードの上下が画面の上下とそのまま対応します（テーブルの裏側や下側が映り込むことはありません）。

**Vキー**、または画面上部の「視点：斜め」「視点：真上」ボタンで切り替えられます（Vキーは文字入力中やメニュー/モーダル表示中は無効化されます）。切り替え時はカメラが0.2〜0.5秒ほどかけて滑らかに移動します。各視点にはそれぞれ適したズーム距離が設定されており、「カメラ初期位置へ戻す」ボタンは常に**現在選択中の視点**の初期位置へ戻ります（斜め視点を選んでいる間は斜め視点の初期位置へ、真上視点を選んでいる間は真上視点の初期位置へ）。選択中の視点は`localStorage`に保存され、再読み込み後も維持されます。

## 山札の操作方法

山札（メインデッキ）を右クリックすると、専用メニューが表示されます。

- **シャッフル** : 山札の順序をランダムに並び替えます（Fisher-Yates法）。
- **1枚ドロー** : 山札の一番上のカードを取り、手札に加えます（表向きになります）。
- **複数枚ドロー** : 枚数を指定するダイアログを表示します（下記）。
- **一番上のカードをめくる** : 山札の一番上のカードを表向きのままテーブル上（山札の隣）へ置きます。手札には加わりません。ドローとは別の操作です。
- **中身を見る** : 山札一覧パネルを開きます（下記）。
- **山札を移動** : 選択後、テーブル上の好きな場所をクリックすると山札全体がその位置へ移動します。

山札が空の場合、「1枚ドロー」「複数枚ドロー」「一番上のカードをめくる」は無効表示になります。

### 山札の中身を見る方法

山札を右クリック →「中身を見る」で、2D一覧パネルが開きます（3D空間には展開されません）。

- 一覧は**上に表示されているカードほど山札の上**にあることを示しています（パネル内にもその旨のヒントを表示）。
- 各行にカード画像・カード名・「上から◯番目 / 全◯枚」という現在位置を表示します。
- 「一番上へ」「一番下へ」ボタンで、その場でカードを山札の一番上/一番下へ移動できます。
- 行をドラッグ＆ドロップすると、その位置に並び替えられます（即座に山札データへ反映され、パネルを閉じても順番は維持されます）。同名カードが複数あってもinstanceIdで区別して正しく並び替えられます。
- 各行の「操作」から、そのカードを「手札へ加える」「フィールドへ表向き/裏向きで出す」「山札の一番上/一番下へ移動」できます。移動したカードは一覧から正しく取り除かれます。
- この一覧UIはカスタム束の「中身を見る」でも共通して使われます。

### 山札内検索

一覧上部の検索欄にカード名を入力すると、部分一致・大文字小文字を区別せずに即時絞り込まれます。検索結果が0件でもエラーにはなりません。検索中は元の山札順を保持しており、検索欄を空にすると全件表示に戻ります。**検索中はドラッグ並び替え・一番上/下へボタンが無効になり**、非表示カードの順番を誤って壊さないようにしています。

### 任意枚数ドロー方法

山札を右クリック →「複数枚ドロー」でダイアログが開きます。「1枚」「3枚」「5枚」のクイックボタン、または枚数入力欄に1以上の整数を指定して「◯枚ドローする」を押します。指定枚数が山札の残り枚数を超える場合は、残り枚数までドローしたうえで通知メッセージが表示されます（エラーにはなりません）。ドローされた順番は保持されたまま手札へ追加され、手札は自動整列します。

## フィールドは自由配置（固定の墓地・除外エリアはありません）

第6段階以降、**固定位置の墓地・除外エリアは廃止**しました。カードゲームによって「捨て札」「除外」に相当するゾーンの意味・置き場所・見た目はまちまちなため、本アプリでは特定のルールを前提とせず、**フィールド上のどこにでも自由にカードを配置する**方式に統一しています。捨て札置き場・除外置き場が必要な場合は、フィールドの好きな場所にカードを表向き/裏向きで置く、または複数枚をまとめて「束にする」（下記「束の作成」を参照）ことで、ゲームごとに好きな運用ができます。

**古いバージョンで保存した盤面データとの互換性**: 墓地・除外エリアが存在した頃に保存された盤面データを読み込んでもエラーにはなりません。読み込み時に自動的に次のように変換されます。

- 墓地にあったカードは、1枚だけならフィールド左側へ、2枚以上なら通常の束（カスタム束）としてフィールド左側へ変換されます。
- 除外エリアにあったカードは、同様にフィールド右側へ変換されます（1枚ならフィールドカード、2枚以上なら束）。
- 変換が行われた場合は、読み込み後に通知メッセージでその旨が表示されます。
- 変換後のカード・束は通常のフィールドカード/カスタム束として扱われ、以後は固定の墓地・除外エリアとして復活することはありません。

## 複数カード選択

フィールド上の通常カード（山札・束の中のカードは対象外）は、Shiftキーを押しながら左クリックすることで複数選択できます。すでに選択中のカードをShift+クリックすると選択から外れます。何もない場所を左クリックするか、Escキーを押すと選択がすべて解除されます。選択状態はUI上だけで管理しており、カードインスタンス自体のデータには保存されません。

## 一括操作

複数選択中に右クリックすると一括操作メニューが表示されます。

- **表向きにする / 裏向きにする / 右へ90度回転 / 左へ90度回転**: 選択カードすべてに適用します。実行後も選択状態を維持します。
- **手札へ戻す**: 選択カードすべてを手札へ移動します。実行後は選択が解除されます。
- **束にする**: 下記「束の作成」を参照してください。
- **選択解除**: 選択をクリアします。

複数選択したカードのどれか1枚をドラッグすると、選択カード全体が相対位置を保ったまま一緒に移動します（テーブル範囲外へは出せません）。

## 束の作成

複数選択した状態で右クリック →「束にする」を選ぶと、選択したカードから新しい束（カスタム束）を作成します。カードが重なっただけでは自動的に束にはなりません。

- 束内のカード順は、選択した順番がそのまま採用されます。
- 束の初期位置は、選択していたカードたちの中心位置になります。
- 束は山札と同じ見た目（重ねて表示、枚数バッジ）で、1つの操作対象として右クリックできます。
- 束の作成後、選択状態は解除されます。

束を右クリックすると専用メニューが表示されます。

- **シャッフル** : 束内のカード順をランダム化します。
- **一番上から1枚取る** : 束の一番上のカードを手札へ加えます。
- **一番上をめくる** : 束の一番上のカードを表向きでフィールドへ出します。
- **中身を見る** : 山札と同じビューアで束の中身を確認・検索・並び替えできます。
- **束を移動** : 選択後、テーブル上の好きな場所をクリックすると束全体が移動します。
- **束を解除** : 下記を参照してください。

メインデッキ・任意の数のカスタム束は、それぞれ別のスタックとしてテーブル上に同時に存在できます。

## 束の解除

束を右クリック →「束を解除」を選ぶと、束の中のカードがフィールド上へ展開されます。

- 元の束の位置付近に、カードが完全に重ならないよう少しずつずらして配置されます。
- 配置順から束の中での順番が視覚的に分かるようになっています。
- 各カードの表裏状態・回転状態は、束にする前の状態がそのまま維持されます。
- 展開後は通常のフィールドカードとして自由に操作できます。

## テーブルリセット方法

画面右上の「テーブルリセット」ボタンを押すと、確認ダイアログが表示されます。「OK」を選ぶと、手札・場・すべてのカスタム束のカードを回収してメインデッキへ戻し、全カードを裏向き・回転0度にしたうえでシャッフルします。カスタム束はこのとき解散されます。「キャンセル」で操作を取り消せます。2人用テストプレイの個別デッキ方式（個別デッキ・同じ構成/別構成）では、各プレイヤーの持ちカードは自分自身の山札にのみ戻り、シャッフルも別々に行われます（相手の山札と混ざることはありません）。

## 盤面の保存方法

画面右上の「盤面を保存」ボタンを押すと、保存名を入力するダイアログが開きます。空の名前では保存できません。既に同じ名前の保存がある場合は上書き確認が表示されます。保存される内容は、使用デッキID・すべてのカードインスタンス・山札/束の順序と位置・手札・選択中のテーブルテーマ・選択中の部屋背景・選択中のカメラ視点（斜め/真上）・カメラ位置と注視点・作成/更新日時、そして2人用テストプレイ中であればプレイヤー情報（名前・現在の手札・自分の山札）・現在のデッキ方式・現在アクティブなプレイヤー・共通デッキのIDも含まれます。Three.jsのオブジェクト自体は保存されず、座標や角度、IDなどの純粋なデータのみが`localStorage`に保存されます。

## 保存した盤面の読み込み方法

「保存した盤面を開く」ボタンを押すと、保存済み盤面の一覧が表示されます。各項目に保存名・使用デッキ名（デッキが削除済みの場合はその旨を表示）・作成日時・更新日時・「読み込む」「名前変更」「削除」ボタンが並びます。「読み込む」を押すと確認ダイアログが表示され、承諾すると現在の盤面を破棄して保存データを復元します（カードの配置・山札/束・手札・テーブルテーマ・部屋背景・カメラ視点・カメラ位置、2人用テストプレイのプレイヤー情報も含めて復元されます）。保存データ内に存在しないカードIDや削除済みデッキが含まれていてもアプリは停止せず、不足分はスキップしたうえで通知が表示されます。**部屋背景を追加する前に保存された古いデータ**（`selectedRoomEnvironmentId`を含まないデータ）を読み込んだ場合はエラーにならず、自動的に「無地」背景として復元されます。**2人用テストプレイ機能を追加する前に保存された古いデータ**（プレイヤー情報を含まないデータ）を読み込んだ場合も、自動的に1人用セッションとして復元されます（プレイヤー交代ボタンや相手の手札表示は現れません）。古い墓地・除外データが含まれる場合の変換については上記「フィールドは自由配置」を参照してください。

## 保存盤面の名前変更・削除

「保存した盤面を開く」の一覧から、各保存の「名前変更」でその場で名前を編集できます（空の名前では変更されません）。「削除」を押すと確認ダイアログの後に削除されます。使用元のデッキを削除しても保存盤面は自動的には削除されません（一覧上で「使用デッキ: （削除済みデッキ）」と表示されるだけです）。

## 2人用オフラインテストプレイ

第6段階で追加した、**1台のPCを2人で交代しながら使う**オフライン専用のパス＆プレイ機能です。ネットワーク通信は一切行わず、前述の「オンライン対戦モード」とは完全に別の仕組みです。既存の1人用テストプレイはこれまで通り利用できます（削除・変更されていません）。

### 始め方

メインメニューの「2人でテストプレイ」を選ぶと設定画面が開きます。

1. プレイヤー1・プレイヤー2の名前を入力します（未入力の場合は「プレイヤー1」「プレイヤー2」が使われます）。
2. デッキ方式を1つ選びます（下記「3つのデッキ方式」を参照）。
3. デッキ方式に応じて、デッキ選択欄が1つ（共通デッキ/個別デッキ・同じ構成の場合）または2つ（個別デッキ・別構成の場合、プレイヤー1用・プレイヤー2用がそれぞれ表示されます）表示されるので、使用するデッキを選びます。カードが1枚も入っていないデッキは選択肢に表示されません。
4. 「テストプレイ開始」を押すと3Dテーブル画面へ遷移します。

### 3つのデッキ方式

- **共通デッキ**: 1つの山札を2人で共有します。ドロー順は1つだけで、山札からドローすると常に**現在アクティブなプレイヤー**の手札に加わります。手札だけがプレイヤーごとに別々に管理されます。
- **個別デッキ・同じ構成**: 同じデッキ構成を2人分（別々のカードインスタンスとして）複製します。カードIDが同じでも、プレイヤー1とプレイヤー2の山札は完全に別物として扱われ、並び順も独立しています。
- **個別デッキ・別構成**: プレイヤーごとに好きな保存済みデッキを個別に選べます。同じデッキを両プレイヤーが選ぶことも可能です。

個別デッキ方式では、フィールド上に「（プレイヤー名）の山札」という2つの山札が別々の位置に表示されます。山札を右クリックした際、**相手の山札に対してはドロー・複数枚ドロー・一番上をめくる・中身を見るが無効化**され、メニューにその旨が表示されます（誤って相手のカードを引いてしまうこと・相手の山札の中身が見えてしまうことを防止するための制限で、シャッフル・山札の移動は引き続き行えます）。

### プレイヤーの交代方法（パス＆プレイ）

画面上部に「現在のプレイヤー：（名前）」と「プレイヤー交代」ボタンが常に表示されます。

1. 「プレイヤー交代」ボタンを押します（**必須の操作方法**です）。Tabキーでも交代できますが、これはあくまで補助的なショートカットで、文字入力欄にフォーカスがある間やメニュー/モーダル表示中は無効です。
2. 画面全体が「（次のプレイヤー名）へ交代します。次のプレイヤー以外は画面を見ないでください。」という確認画面に切り替わり、それまでの手札は完全に隠されます。
3. 次のプレイヤーが「手札を表示」ボタンを押すと確認画面が閉じ、そのプレイヤー自身の手札が表示されます。

交代前のプレイヤーの手札は交代した瞬間に保存され、交代後は新しいアクティブプレイヤーの手札だけが画面下部の手札パネルに表示されます。山札からのドロー・手札からフィールドへの配置などは、常にその時点でアクティブなプレイヤーの手札を対象に行われます。

### 相手の手札表示

自分がアクティブでない方のプレイヤーの手札は、フィールド上部に「（名前） / 裏向きの小さいカードが並んだ表示 / 手札 ◯枚」という形でカードの**枚数だけ**表示されます。中身（カードの絵柄・種類）は一切表示されず、クリックしても内容は見えません。手札が0枚のときも「手札 0枚」ときちんと表示されます。プレイヤーを交代すると、表示される「相手」も自動的に入れ替わります。

### 1人用テストプレイとの違い

1人用テストプレイでは、上記のプレイヤー切替UI・相手の手札表示・デッキ方式選択は一切表示されません（従来通りの画面のままです）。2人用テストプレイのために追加された仕組みは、2人用セッションでのみ有効になります。

## プレイヤーごとのカード向きとデッキ位置

第7段階で追加した、2人用テストプレイ専用の改善です。

- **カードの初期向き**: 手札から場に出したカード（山札の一番上をめくる操作も含む）は、出した瞬間の**現在アクティブなプレイヤー**に応じて初期角度が決まります。プレイヤー1が出すと0度、プレイヤー2が出すと180度で配置され、向かい合う2人がそれぞれ自分から見て正しい向きでカードを読めます。1人用テストプレイでは常に0度で、これまでと見た目は変わりません。
- 配置後にプレイヤーを交代しても、既に置かれているカードの向きが勝手に変わることはありません。右クリックメニューからの手動回転（90度単位）はこれまで通り行えます。
- 手札へ戻したカードを別のプレイヤーが再度場に出した場合は、その時点でアクティブなプレイヤーの向きが新たに適用されます。
- カードの向きは盤面の保存・読み込みでもそのまま維持されます。
- **デッキの初期位置**: 個別デッキ方式（同じ構成/別構成）では、各プレイヤーの山札はそれぞれの座席から見て**右側**に初期配置されます（プレイヤー1は手前から見て右、プレイヤー2は奥から見て右 - 向かい合っているため、ワールド座標上は左右反対側になります）。共通デッキ方式では、これまで通り中央に1つだけ配置されます。
- 山札は初期配置後、自由に移動できます。盤面を保存・読み込みした場合は、保存時の位置がそのまま復元されます（勝手に初期位置へ戻ることはありません）。

## 裏向きカードの非公開確認

第7段階で追加した機能です。手札から裏向きで場に出した（または右クリックで裏向きにした）カードは、盤面上ではそのカードを出した/裏向きにしたプレイヤーだけが、周りにばれずに中身を確認できます。

- フィールド上の裏向きカードを右クリックすると、カードメニューに「自分だけ確認」が表示されます。
- 確認できるのは、**そのカードを最後に裏向きにした（または裏向きで場に出した）プレイヤー**だけです。1人用テストプレイでは常に確認できます。確認権限がない場合は「このカードは確認できません」という通知が表示されます。
- 「自分だけ確認」を選ぶと、右側のカードプレビューが「自分だけ確認中」というラベル付きでそのカードの表面を表示します。**盤面上のカード自体は裏向きのまま**で、`faceUp`の状態が変化することはありません。
- 確認画面は「閉じる」ボタンで手動で閉じられるほか、別のカードを選択したとき・プレイヤーを交代したとき・盤面を保存/読み込みしたときは自動的に閉じられます。確認状態は一時的なもので、盤面の保存データには含まれません。

## カードプレビューのルーペ

第7段階で追加した機能です。右側のカードプレビュー画像の上でマウスを動かすと、Amazonなどの商品画像のように、マウス位置を中心に約2.5倍で拡大表示する虫眼鏡（ルーペ）が追従します。画像の外へマウスが出ると通常表示に戻ります。元のカード画像を拡大コピーして表示しているだけなので、画質が劣化することはありません。設定画面からON/OFFを切り替えられ、初期状態はONです（詳細は下記「設定」を参照）。

## UIパネルの折りたたみと集中モード

第7段階で追加した機能です。テストプレイ画面の主要な4つのUI（左側の操作方法パネル、右側のカードプレビュー、上部の操作バー、下部の手札パネル）は、それぞれ独立して折りたためます。

- 各パネルの右上・左上などにある折りたたみボタンで開閉できます。折りたたむと小さいタブ状のボタンだけが残り、そこから再度開けます。
- 開閉状態はそれぞれ`localStorage`に保存され、次回起動時も維持されます。

**集中モード**は、上記4つのパネルをまとめて最小化し、3Dフィールドをできるだけ広く使うためのモードです。

- 画面上部の「集中モード」ボタンで開始します。開始すると、他の操作ボタンはすべて隠れ、「集中モード解除」ボタンだけが残ります。
- 集中モード中も、カードの選択・移動・右クリックメニューなどの操作はこれまで通り行えます。
- 「集中モード解除」ボタン、またはEscキー（選択解除より優先されます）で通常表示に戻ります。戻ると、集中モードに入る直前の各パネルの開閉状態がそのまま復元されます。
- 集中モードそのものは一時的な状態としてのみ扱われ、`localStorage`には保存されません（起動するたびに解除された状態から始まります）。

## カスタムプレイマット

第7段階で追加した機能です。設定画面から、自分の画像をプレイマットとして使用できます。

- 対応形式は **PNG / JPG / JPEG / WebP** です。ファイルサイズの上限は5MBで、これを超える画像や対応形式以外の画像は分かりやすいエラーメッセージとともに拒否されます。
- 設定画面の「プレイマット画像を追加」から画像を選ぶと、登録済みプレイマットの一覧に追加され、自動的に選択状態になります。
- 登録済みプレイマットは一覧からいつでも選び直せるほか、削除もできます。「標準プレイマット」を選ぶと、これまで通りテーブルテーマに連動した生成テクスチャへ戻ります。
- 縦横比がテーブルと異なる画像は、歪ませずに中央部分をトリミングして敷き詰めます（CSSの`object-fit: cover`に近い扱いです）。
- 画像データは**ブラウザ内のIndexedDBにのみ**保存され、外部へ送信されることはありません。プレイマットを切り替えても、フィールド上のカードが消えることはありません。
- 選択中のプレイマットは`localStorage`に保存され、次回起動時も復元されます。

## テーブルテーマ・部屋背景の変更方法

画面右上の「テーブル設定」ボタンを押すと、「テーブル」「部屋」の2つのタブを持つ設定パネルが開きます。どちらのタブでも、プレビュー画像・説明文・現在選択中の表示を見ながら「選択」を押すと即座に3D空間へ反映されます（保存ボタンは不要です）。パネルは3Dテーブルを完全には隠さない大きさに収まっています。選択内容はそれぞれ独立して`localStorage`に保存され、再読み込み後も維持されます。

**テーブルテーマと部屋背景は完全に独立して選択できます**（例: クラシックグリーンのテーブル＋おうち、赤いじゅうたんのテーブル＋カジノ、など自由に組み合わせられます）。一方を変更してももう一方は変わりません。

### テーブルテーマ

- **赤いじゅうたん** : 深い赤の布・じゅうたん風。高級感のある配色。
- **クラシックグリーン** : 落ち着いた緑のフェルト風（初期状態のデフォルトテーマ）。
- **ダークテーブル** : 濃いグレーに控えめな木目風パターン。

いずれもCSS/Canvasでその場生成した模様とグラデーションのみで構成しており、外部のテクスチャ画像は使用していません。第5段階で全体的に明るさを見直し、以前より視認性が高くなっています。

### 部屋背景

テーブルの周囲には部屋の床・壁が常に表示され、カメラをどこに動かしてもテーブルの外側が何もない空間にならないようになっています。第7段階で、どのテーマにも共通の巾木（床と壁の境目の縁取り）・ドア・窓・天井照明を追加し、空箱に見えないよう底上げしました（天井そのものは、大きな水平の天井板がこの環境のソフトウェアWebGLフォールバックで描画を壊すことが分かっているため、あえて設置していません）。用意されている4種類の部屋背景から選べます。

- **無地** : 明るすぎないグレーの床と壁だけのシンプルなスタジオ背景。装飾がなく最も軽量で、カードとテーブルが最も見やすい標準背景です。
- **おうち** : 木目の床・明るい壁・窓・棚・観葉植物・簡易的なソファのシルエットがある、暖かく落ち着いたリビング風の部屋です。
- **カジノ** : 深い赤のカーペット・暗めの壁・間接照明の光る帯・金色のトリム・奥に見える別テーブルのシルエットがある、架空のカードルームです（実在するカジノのロゴやブランドは使用していません）。
- **カードショップ** : 明るい床・白い壁・抽象的な色ブロックで表現したカード棚と商品・ショップサイン・レジカウンター・ポスター風パネル・椅子付きの対戦テーブルが複数ある、明るいカードショップの対戦スペースです（実在する商品画像やロゴは使用していません）。カード商品の棚（24個の箱）はInstancedMesh（1回の描画命令で大量の同じ形状を描く仕組み）でまとめて描画しており、見た目を保ったまま描画負荷を抑えています。

いずれもThree.jsの基本形状（板・箱・円柱・球）と単色マテリアルのみで構成しており、外部のテクスチャ画像や3Dモデルファイルは使用していません。「無地」が最も軽量な構成で、低スペックなPCでもテーブル操作のフレームレートを優先できます。背景の装飾オブジェクトはいずれもカード操作用のRaycast（クリック判定）から除外されており、カードの選択・ドラッグを妨げることはありません。

## 3D品質設定

第7段階で追加した機能です。設定画面から3D描画の品質を選べます。

- **標準** : 通常の影（PCFシャドウマップ）、各部屋テーマの装飾オブジェクトをすべて表示、通常の照明。
- **軽量** : 影を無効化し、各部屋テーマの装飾オブジェクト（ポスターや一部のテーブルなど）を間引いて表示します。低スペックなPCで操作性を優先したい場合に選んでください。

品質を切り替えてもカードの配置やゲームの状態が初期化されることはありません。切り替え時に不要になったGeometry/Material/Textureは、React（React Three Fiber）のアンマウント処理により自動的に破棄されます。選択した品質は`localStorage`に保存され、次回起動時も維持されます。

## 新しいカード画像の追加方法

1. `src/assets/cards/` フォルダに **PNGまたはJPEG(.jpg/.jpeg)形式** の画像を追加します（例: `card-005.png`）。
2. カード裏面は `src/assets/card-back.png`（または `.jpg`/`.jpeg`）を差し替えます（全カード共通の1枚）。
3. `npm run dev` を再起動する（またはVite開発サーバーがホットリロードで検知）と、BOX画面に自動で一覧表示されます。

- 対応形式は **PNG / JPEG** です。AI/PSD/PDFには対応していません。
- カードIDは画像パス（ファイル名）から自動生成され、BOX画面でカード名を変更してもIDは変わりません。
- カード画像フォルダが空でもアプリはクラッシュせず、「カードが見つかりません」という表示になります。

## 新しいテーブルテーマの追加方法

`src/config/tableThemes.ts` の `TABLE_THEMES`配列にオブジェクトを1つ追加するだけで登録できます。

```ts
{
  id: 'my-theme',
  name: '好きな名前',
  tableColor: '#334455',       // 表面の基調色
  surfacePattern: 'felt',      // 'weave' | 'felt' | 'planks' から選択
  backgroundColor: '#05060a',  // 3D空間の背景色
  previewPath: swatchDataUri('#334455', '#05060a'), // テーマ一覧用のプレビュー画像
}
```

`surfacePattern` は `src/lib/tableTextureGenerator.ts` がCanvasでその場生成する模様の種類です（weave=じゅうたん風の織り目、felt=フェルトの微細なノイズ、planks=控えめな板目）。新しい模様を追加したい場合はこのファイルに描画関数を追加してください。

## 新しい部屋背景の追加方法

`src/config/roomEnvironments.ts` の `ROOM_ENVIRONMENTS`配列にオブジェクトを1つ追加し、`src/components/table/environments/` に対応するコンポーネントを1つ追加します。

```ts
// src/config/roomEnvironments.ts に追加
{
  id: 'my-room',
  name: '好きな名前',
  description: '一覧に表示される説明文',
  previewPath: roomSwatchDataUri('#334455', '#556677', '#05060a'), // 床色・壁色・背景色
  floorColor: '#334455',
  wallColor: '#556677',
  backgroundColor: '#05060a',
  ambientLightIntensity: 0.9,   // 環境光の強さ
  directionalLightIntensity: 1.3, // メインライトの強さ
}
```

```tsx
// src/components/table/environments/MyRoomEnvironment.tsx
import type { RoomEnvironment } from '../../../config/roomEnvironments';
import { RoomShell } from './RoomShell';

export function MyRoomEnvironment({ environment }: { environment: RoomEnvironment }) {
  return (
    <RoomShell floorColor={environment.floorColor} wallColor={environment.wallColor}>
      {/* boxGeometry/cylinderGeometryなど基本形状で装飾を追加 */}
    </RoomShell>
  );
}
```

最後に `src/components/table/environments/RoomEnvironmentRenderer.tsx` の分岐に新しいIDを追加すれば、テーブル設定パネルの「部屋」タブに自動的に一覧表示されます。`RoomShell`が床・壁の共通部分（テーブルが常に部屋の中にあるように見せるための土台）を担当するため、新しい背景では家具などの装飾だけを追加すれば済みます。

## オンライン対戦モード

第4段階で追加した、**2人用・ローカル動作限定**のミニマルなオンライン対戦機能です。オフラインのテストプレイとは完全に独立した画面・データで動作し、既存のオフライン機能（BOX・デッキ編集・盤面保存など）には一切影響しません。

### 概要と制限

- 現時点では**2人用専用**です（3人目が参加しようとすると「ルームが満員です」と表示され拒否されます）。3人以上・観戦・チャット・ログイン・パスワード付きルーム・公開ルーム一覧・マッチングには対応していません。
- ルーム状態は**サーバーのメモリ上のみ**に保持されます。データベースやファイルへの永続化は一切行っておらず、**サーバーを再起動するとすべてのルームが消えます**（意図した仕様上の制限です）。
- 盤面の名前付き保存/読み込み（`localStorage`機能）はオンラインモードでは使用しません（オフラインの盤面保存/読み込みはこれまで通りオフライン専用画面でのみ利用できます）。
- カメラの位置・向きは各プレイヤーのブラウザだけで管理するローカルな情報で、サーバーには送信されず、相手とは共有されません。
- ターン管理・勝敗判定などのルール処理は行いません（オフラインモードと同様）。
- カードのロック（同時操作の排他制御）は未実装です。同時に同じカードを操作した場合は「サーバーが受け取った順で後勝ち」というシンプルな方針で処理します。

### ルームの作成・参加方法

1. メインメニューの「オンライン対戦」→「オンラインルームを作成」から、プレイヤー名（1〜20文字）・使用デッキ・テーブルテーマを選んでルームを作成します。ルームIDは`XXXX-XXXX`形式（例: `A7K9-PQ2M`。`0`/`O`/`1`/`I`など紛らわしい文字は使われません）でランダムに生成されます。
2. ルーム作成者に特別な権限はありません。参加者は全員対等です。
3. ルーム画面右上の「招待URLをコピー」で招待URL（`http://<ホスト>/room/<ルームID>`）をコピーできます。Clipboard APIが使えない環境では、代わりにテキスト欄が表示され手動でコピーできます。
4. 2人目のプレイヤーは、招待URLを直接開くか、「オンライン対戦」→「オンラインルームに参加」からルームIDを直接入力して参加します。
5. 存在しないルームID・満員のルーム・不正な形式のルームIDに対しては、それぞれ分かりやすいエラーメッセージが表示されます。

### プレイヤー名とID

- プレイヤー名は1〜20文字（前後の空白は自動で除去）で、未入力では参加できません。名前は簡易サニタイズ（`< > & " ' `を除去）を通してから使用され、HTMLとして解釈されることはありません。
- 参加者を実際に区別する内部IDは`playerId`で、初回アクセス時にブラウザ側で生成し`localStorage`（キー: `vct.online.playerId.v1`）に保存されます。プレイヤー名はUI表示専用で、`playerId`が本当の同一性の判定に使われます。

### 同期される操作

サーバーが状態を一元管理し（サーバー権威型）、クライアントは操作をサーバーへ送信 → サーバーが検証・反映 → 全参加者へ配信、という流れで動きます。以下の操作が両プレイヤー間で同期されます。

- カードの移動（ドラッグ）・表裏切替・90度回転
- 手札⇔フィールドの移動、墓地へ送る、除外する、山札の上/下へ戻す
- 山札のシャッフル・1枚ドロー・任意枚数ドロー
- カスタム束の作成・解除・移動、束内の並び替え
- テーブルリセット、テーブルテーマの変更
- プレイヤーの参加・退出・接続状態

ドラッグ中のカード位置は相手にもリアルタイムで見えますが、通信量を抑えるため約80msごとに間引いて送信し、ドロップ時に確定情報を1回送信します（確定情報には更新順を示す情報を含み、古い情報が新しい情報を上書きしないようにしています）。

### 手札の非公開について

自分の手札は自分だけが本当の絵柄を見ることができ、相手には「枚数」と「裏面」だけが見えます。これは**見た目を隠しているだけではなく、サーバーが相手のクライアントには元々カードの正体を送信しない**方式で実現しています。具体的には、サーバーは参加者ごとに内容を作り分けた状態を個別に送信しており、自分が持たない手札カードについては`cardId`（カードの正体を示す情報）が常に`null`として届きます。ブラウザの開発者ツールで通信内容を見ても、相手の手札の正体は分かりません。

山札の中身についても、シャッフル後の並び順が全員に公開されることはなく、通常は「残り枚数」だけが公開されます。「中身を見る（自分だけ確認）」は、要求した本人にだけ内容を返す専用の応答として実装されており、他の参加者に配信されることはありません。

手札カードをフィールドへ出した後、そのカードをもう一度手札へ戻す操作をすると、**元の持ち主の手札へ**戻ります（操作した人の手札に入るわけではありません）。持ち主が不明な場合は操作が拒否されます。

### 再接続について

ページを再読み込みしたり、一時的に接続が切れたりした場合は、保存済みの`playerId`とURL中のルームIDを使って自動的に元のルームへ再接続を試みます。再接続に成功すると、自分の手札を含めた最新の盤面が復元され、相手の画面上でも自分の接続状態が「オンライン」に戻ります。同じ`playerId`で2つ目のブラウザ/タブから接続した場合は、後から接続した方が有効になり、古い方の接続は無効になります。

参加者一覧には各プレイヤーの名前・接続状態（オンライン/オフライン）・自分自身であることを示す表示が出ます。

### 空room（誰もいなくなったルーム）の扱い

全員が退出した空のルームは、最後の1人が抜けてから**1時間**はサーバーのメモリ上に保持されます（数分おきに期限切れのルームをまとめて破棄する処理が動いています）。1時間を過ぎたルーム、またはサーバーが再起動した場合は、そのルームの盤面はすべて失われます。

### 動作確認方法

このモードはローカル環境専用で、外部のクラウドには一切デプロイしていません。動作確認は次の方法で行えます。

1. `npm run dev:online` でクライアントとサーバーを起動します。
2. 通常のブラウザウィンドウでルームを作成します。
3. 別のブラウザ（シークレットウィンドウ、別ブラウザ、または同一LAN内の別端末）で招待URLを開き参加します（同じブラウザ・同じ`localStorage`だと同一プレイヤー扱いになってしまうため、必ずプロファイルを分けてください）。
4. 同一LAN内の別端末から確認する場合は、サーバーが `PORT`（既定`3001`）でLAN内から到達可能であることを確認したうえで、クライアント側の接続先ポートは環境変数 `VITE_ONLINE_SERVER_PORT` で上書きできます（クライアントは`window.location.hostname`を使ってサーバーへ接続するため、`localhost`固定にはなっていません）。

## メインメニュー

第7段階でデザインを刷新しました。黒~濃紺を基調に、半透明のガラス風パネル・細い枠線・控えめな発光を使ったオリジナルデザインです（特定の既存カードゲーム作品のUIを模したものではありません）。

- 背景には実際の3Dテーブル・部屋（現在選択中のテーマ、無地の部屋を基準に軽量設定で描画）をそのまま再利用しています。カメラはごく小さくゆっくり動くだけで、カード操作はできず、Raycast（クリック判定）の対象にもなりません。画面を離れると3D描画は通常通りアンマウントされ、不要なアニメーションループが残ることはありません。
- タイトルロゴのフェードイン、ボタンのホバー時の浮き上がりなど軽量な演出を加えていますが、パーティクルや動画背景、長い起動演出はありません。`prefers-reduced-motion`が有効な環境ではアニメーションを無効化します。
- **1人でテストプレイ**が最も目立つ主要ボタンです。保存済みの盤面がある場合のみ、**前回の続きから**ボタンが並んで表示され、直前に保存した盤面をそのまま再開できます（保存データがない場合は表示されません）。
- **2人でテストプレイ**もメイン操作として並びます。
- **BOX**・**デッキ編集**・**オンライン対戦**・**設定**はサブ操作としてまとめて表示されます。

## 設定

第7段階で追加した設定画面です（メインメニューの「設定」から開きます）。以下の項目をまとめて管理します。

- カードプレビューのルーペのON/OFF
- 3D品質（標準/軽量）
- カスタムプレイマットの追加・選択・削除

いずれも`localStorage`（プレイマット画像本体のみIndexedDB）に保存され、次回起動時も復元されます。古いバージョンのデータにこれらの項目が存在しない場合は、安全な初期値（ルーペON・品質は標準・プレイマットは標準）が使われるため、既存の保存データとの互換性は保たれます。

状態管理は6系統のZustandストアに分離しています。

- `useCardMasterStore` : カード原本データ + 名前上書き（`localStorage`永続化）
- `useDeckStore` : デッキのCRUD（`localStorage`永続化）
- `useTableStore` : オフラインのテストプレイ中のカードインスタンス（`Record<instanceId, CardInstance>`）、山札/束（`CardStack[]`）、**現在アクティブなプレイヤーの**手札順（`string[]`）、選択（複数）/ドラッグ/各種右クリックメニュー/山札ビューアの状態、選択中テーブルテーマ・選択中の部屋背景・選択中のカメラ視点（`oblique`/`top`）・手札パネルの開閉状態（`localStorage`永続化、盤面自体は非永続）、手札→フィールドのドラッグ中プレビュー位置と表裏の向き（非永続の一時状態）、2人用テストプレイのプレイヤー一覧・現在のデッキ方式・共通デッキID・プレイヤー交代の確認画面表示状態（`players`/`currentPlayerIndex`/`testPlayMode`/`sharedDeckStackId`/`awaitingHandReveal`、いずれも非永続）、裏向きカードの一時確認状態（`peekingInstanceId`、非永続）、左右上部/手札パネルの折りたたみ状態と集中モード（`leftPanelCollapsed`/`previewPanelCollapsed`/`topBarCollapsed`は`localStorage`永続化、`focusMode`は非永続）、ルーペ有効/3D品質/選択中プレイマットID（`loupeEnabled`/`graphicsQuality`/`selectedPlaymatId`、いずれも`localStorage`永続化）
- `useSavedBoardsStore` : 名前付き盤面保存データの一覧・CRUD（`localStorage`永続化、オフライン専用）
- `useUiStore` : 検索欄の入力値・通知バナーなど画面固有のUI状態
- `useOnlineStore` : オンラインルームの接続状態・参加者一覧・サーバーから受信した盤面（`PublicTableState`、常にサーバーが権威）・ドラッグ中のローカル座標オーバーライドなど

3Dオブジェクト自体はストアに保存せず、座標・回転・表裏・所属ゾーン（`deck`/`hand`/`table`、`graveyard`/`banished`は廃止済みで旧データ読み込み時の変換にのみ使用）などの純粋なデータのみを保存しているため、オフラインとオンラインでデータ層とレンダリング層を分離したまま両立できています。状態変更はすべて各ストアのアクションに集約しており、3Dコンポーネント側で複数のストアを直接書き換えることはありません。オンラインモードでは`useTableStore`を一切使わず、`useOnlineStore`が完全に独立した状態を持つため、オフラインの盤面データとオンラインの盤面データが混ざることはありません。2人用テストプレイでも、アクティブなプレイヤーの手札は引き続き`hand`フィールドがそのまま保持し（既存の手札操作アクションはすべて無変更で動作します）、プレイヤー交代の瞬間だけ`players`配列との間で手札を退避/復元する設計になっています。

## 現在の制限事項

- ルール処理（ターン、フェーズ、コスト、ダメージ、勝敗判定など）は一切ありません。
- 山札・束の中身は「一覧を見て個別に移動する」形式で操作します。中身を見たままドラッグで直接並べ替える以上の自由な検索操作（例: 条件抽出）はありません。
- フィールド上の複数選択は矩形（範囲）ドラッグには対応していません（Shiftクリックのみ）。
- 盤面の保存・読み込みは`localStorage`内のみで完結します（サーバー保存・クラウド保存はありません）。
- 固定の墓地・除外エリアはありません（フィールドへの自由配置、または束にまとめる方式に統一しています）。
- オフラインの2人用テストプレイは**同一PCでの交代制（パス＆プレイ）専用**です。同時に2人分の入力を別々の端末から行うことはできません（複数端末からの同時操作が必要な場合はオンライン対戦モードをご利用ください）。
- オフラインの2人用テストプレイは**3人以上のプレイヤーには対応していません**。
- 2人用テストプレイのプレイヤー数・デッキ方式は、テストプレイ開始時にのみ選択でき、開始後にモードを変更する機能はありません（改めて設定画面からやり直す必要があります）。
- サイコロ・コイン・トークン・カウンターなど補助要素はありません。
- 物理演算は使用していません（座標を直接計算・管理しています）。
- スマートフォン対応はしていません（PCブラウザ専用）。
- カード画像の差し替え・追加はファイルシステムへの手動配置のみで、UIからのアップロードはできません。
- オンライン対戦モードは**2人用のみ**です。3人以上・観戦・チャット・ログイン・パスワード付きルーム・公開ルーム一覧・マッチング・フレンド機能はありません。
- オンラインのルーム状態はサーバーのメモリ上のみで管理しており、データベース・クラウドストレージへの永続化やサーバー再起動をまたいだ復元はありません。
- オンラインモードでの盤面の名前付き保存/読み込み・エクスポート/インポートはありません（オフラインの盤面保存/読み込みはこれまで通り利用できます）。
- オンラインでのカード操作にロック（排他制御）はありません。同時操作は「サーバー受信順で後勝ち」で解決します。
- オンラインでのUndo/redo・操作履歴・観戦モード・管理者権限はありません。
- 本番環境（外部クラウド）へのデプロイは行っておらず、想定していません（ローカル/LAN限定）。
- 部屋背景は用意された4種類から選ぶのみで、家具の自由配置・部屋の編集・部屋サイズの変更・背景画像や3Dモデルのアップロードには対応していません。
- 椅子に座る・一人称視点・キャラクターアバター・VR対応はありません。
- 裏向きカードの「自分だけ確認」は、そのカードを最後に裏向きにした/裏向きで場に出したプレイヤーのみが行える単純な仕組みです。カードの所有権・秘匿情報を管理する本格的な仕組みではありません。
- カードプレビューのルーペはマウス操作専用です（タッチ操作では動作しません）。
- カスタムプレイマットは1枚の画像をテーブル全体に敷くだけで、複数画像の合成や部分的な貼り付けはできません。ファイルサイズ上限は5MB、対応形式はPNG/JPG/JPEG/WebPのみです。
- 3D品質設定は「標準」「軽量」の2段階のみで、影の解像度や個々のライトを細かく調整することはできません。
- メインメニューの3D背景は現在選択中のテーブルテーマを使用しますが、部屋背景は常に「無地」で固定表示されます（軽量な背景表示を優先しているためです）。

## 素材・ライセンスに関する注意

- テーブルの表面模様・背景・READMEサムネイルは、すべてこのプロジェクト内でCSS/Canvas/SVGにより生成したオリジナル素材です。外部の有料素材・既存ゲームの画像・既存ゲームのUIは使用していません。
- 同梱されているサンプルカード画像（`card-001.png`〜`card-004.png`、`card-back.png`）はこのプロジェクト用に生成したプレースホルダーです。実際のカードゲーム制作物では、著作権上問題のない自作画像に差し替えてください。
- 外部素材を追加する場合は、商用利用・改変・再配布が可能でクレジット表記条件を満たすものか、必ず事前に確認してください。

## 既知の不具合

- 右クリックで「カードを掴んだままドラッグして視点回転しようとする」操作をすると、カード側の`contextmenu`イベントが先に発火しメニューが開くことがあります（右ドラッグでの視点回転は、カードのない場所から始めてください）。
- デッキ編集画面で「保存」を押す前に別デッキへ切り替える、または画面を離れると、未保存の変更は失われます。
- 「テーブルから取り除く」で削除したカードは完全に削除されるため、「テーブルリセット」や盤面の保存/読み込みでも復元されません。
- 山札の一番上を続けてめくる／複数枚ドローすると、フィールドへ出た複数のカードが同じ既定位置に重なって出ることがあります（それぞれ自由に位置を調整できます）。
- 大量のカード画像・大きなデッキ（100枚超規模）を読み込んだ場合のパフォーマンスは簡易的な確認のみです。
- オンラインモードで山札からカードを引いた直後など、そのカードの画像テクスチャの読み込みが完了する前に極端に早いタイミングで見ると、画像が読み込まれるまでの一瞬だけプレースホルダー色で表示されることがあります（読み込みが完了すれば自動的に正しい絵柄に切り替わります）。
